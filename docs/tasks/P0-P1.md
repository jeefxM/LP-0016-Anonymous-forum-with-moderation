## Phase 1 — Crypto spike

### P1.1 — Lock in deps for the post-proof guest

- [ ] **Acceptance:** `programs/post_proof/methods/guest/Cargo.toml` pins
      `risc0-zkvm` (no `std`, in guest), `serde`, `poseidon-rs` (or similar
      RISC0-compatible Poseidon impl — verify in this task).
- [ ] **Verify:** `cargo risczero build --manifest-path
      programs/post_proof/methods/guest/Cargo.toml` succeeds.
- [ ] **Files:** `programs/post_proof/methods/guest/Cargo.toml`,
      `programs/post_proof/methods/guest/src/bin/post_proof.rs` (skeleton).

### P1.2 — Guest skeleton: commitment + nullifier + share

- [ ] **Acceptance:** Guest reads `(secret, content_id, epoch)` private and
      `(tree_root, epoch, content_id)` public, computes
      `commitment = poseidon(secret)`,
      `nullifier = poseidon(secret, epoch)`,
      `share_y = secret_lower + poseidon(secret, content_id) * nullifier`,
      and `env::commit`s `(commitment, nullifier, share_y)`.
- [ ] **Verify:** Local host run produces a journal that decodes to expected
      values for a fixed (secret, content_id, epoch).
- [ ] **Files:** `programs/post_proof/methods/guest/src/bin/post_proof.rs`,
      `crates/proof-host/src/lib.rs` (minimal runner).

### P1.3 — Merkle membership inside guest

- [ ] **Acceptance:** Guest takes a Merkle path (siblings + indices) as
      private input and verifies it against the public `tree_root`. Panics on
      mismatch. Depth = 16 (supports 2^16 = 65,536 members per instance).
- [ ] **Verify:** Two unit tests — happy path (in-tree) succeeds, tampered
      sibling fails with `ProgramExecutionFailed`.
- [ ] **Files:** Same as P1.2 + a small `merkle.rs` helper module.

### P1.4 — End-to-end benchmark with `RISC0_DEV_MODE=0`

- [ ] **Acceptance:** Host binary `bench-post-proof` generates a real proof
      with dev mode off and reports wall-clock time. Run on the dev MacBook.
- [ ] **Verify:** `RISC0_DEV_MODE=0 cargo run --release --bin bench-post-proof`
      reports a number. Capture stdout in `docs/adr/ADR-002-risc0-feasibility.md`.
- [ ] **Files:** `crates/proof-host/src/bin/bench_post_proof.rs`.
- [ ] **Hard limit:** if &gt;10 s, stop the line and re-plan (open ADR-002 with
      pivot options).

### P1.5 — ADR-002: proceed or pivot

- [ ] **Acceptance:** Documents benchmark number, decision, and (if pivot
      needed) the alternative plan.
- [ ] **Verify:** `cat docs/adr/ADR-002-risc0-feasibility.md`
- [ ] **Files:** `docs/adr/ADR-002-risc0-feasibility.md`.
