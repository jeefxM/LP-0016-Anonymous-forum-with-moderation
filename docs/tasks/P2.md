# Tasks: Phase 2 — LEZ membership registry program

> Goal: a deployable LEZ program with `Register` and `Slash` (stub).
> Slash signature/share verification stays a `panic!` until P5.

## Design recap

**One account per forum instance** — a PDA derived from `forum_id`. Holds:

```
ForumState {
    tree_root: [u8; 32],                  // commitments Merkle root (depth 16)
    next_leaf_index: u32,                  // next free slot in the tree
    revocation_set: Vec<[u8; 32]>,         // slashed commitments
    config: ForumConfig {
        k_threshold: u8,                   // strikes → slash
        n_threshold: u8,                   // N-of-M for cert issuance
        moderator_pubkeys: Vec<Ed25519Pub>,// M
        stake_amount: u128,                // GLM per member
    },
}
```

**Register flow** — caller pre-computes incremental Merkle path; guest
verifies path against `tree_root` with leaf=0 at `next_leaf_index`, then
re-hashes with leaf=commitment, writes new root + increments index.

**Slash flow (stub)** — guest panics `"slash not implemented"` so the
instruction shape is real but verification waits for P5.
